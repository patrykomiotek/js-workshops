Czym jest JavaScript?

  • Język programowania stworzony w 1995 roku
  • Autor: Brendan Eich (Netscape)
  • Powstał w zaledwie 10 dni
  • Pierwotna nazwa: Mocha, potem LiveScript

Brendan Eich stworzył prototyp w 10 dni. Nazwa JavaScript to chwyt marketingowy - Java była wtedy bardzo popularna. JS nie ma prawie nic wspólnego z Javą.

JavaScript w testowaniu

Popularne narzędzia testowe oparte na JS:

Cypress


it("should add product to cart", () => {
  cy.visit("/products");
  cy.get(".add-to-cart").click();
  cy.get(".cart-count")
    .should("contain", "1");
});

Playwright


test("search for product", async ({ page }) => {
  await page.goto("/");
  await page.fill("#search", "shampoo");
  await page.click(".search-btn");
  await expect(page.locator(".results"))
    .toBeVisible();
});
Pokazać, że znajomość JS jest kluczowa dla inżynierów testów automatyzujących. Cypress i Playwright to 100% JavaScript. k6 używa JS do definiowania testów wydajnościowych. Nie trzeba rozumieć każdej linii - chodzi o zobaczenie, że JS jest wszędzie w testach.

k6 - testy wydajnościowe


import http from "k6/http";
import { check } from "k6";

export default function () {
  const response = http.get("https://api.rossmann.pl/products");

  check(response, {
    "status is 200": (r) => r.status === 200,
    "response time < 500ms": (r) => r.timings.duration < 500,
  });
}
k6 to narzędzie do testów wydajnościowych napisane w Go, ale skrypty pisze się w JS. Znajomość JS pozwala pisać zaawansowane scenariusze obciążeniowe.

Zmienne

Przechowywanie danych w programie

Zmienne to jeden z najważniejszych konceptów. Bez nich nie można pisać programów. Będziemy uczyć się let, const i var.

Czym jest zmienna?

Zmienna to pudełko z etykietą, w którym przechowujemy dane. 


let productName = "Szampon do wlosow";
  • let - słowo kluczowe (tworzymy zmienną)
  • productName - nazwa (etykieta na pudełku)
  • = - operator przypisania (wkładamy do pudełka)
  • "Szampon do wlosow" - wartość (zawartość pudełka)
Analogia z pudełkiem jest świetna dla początkujących. Etykieta to nazwa zmiennej, zawartość to wartość. Możemy zmienić zawartość, ale etykieta zostaje.

let - deklaracja zmiennej


// Deklaracja i przypisanie
let price = 29.99;
console.log(price); // 29.99

// Zmiana wartosci
price = 24.99;
console.log(price); // 24.99

// Deklaracja bez przypisania
let discount;
console.log(discount); // undefined

Zapamiętaj

let pozwala zmienić wartość zmiennej później.

let to nasz główny sposób na tworzenie zmiennych. Wartość można zmienić. Jeśli nie przypiszemy wartości, zmienna ma wartość undefined.

const - stała


const companyName = "Rossmann";
console.log(companyName); // "Rossmann"

// Proba zmiany - BLAD!
companyName = "Inna firma";
// TypeError: Assignment to constant variable.

Uwaga

const wymaga przypisania wartości przy deklaracji! 


const taxRate; // SyntaxError!
const taxRate = 0.23; // OK
const to stała - raz przypisana wartość nie może być zmieniona. Używamy const domyślnie, a let tylko gdy wiemy, że wartość się zmieni. To dobra praktyka - chroni przed przypadkowymi zmianami.

var - stary sposób


var userName = "Anna";
console.log(userName); // "Anna"

// var pozwala na re-deklaracje (let nie!)
var userName = "Bartek"; // OK z var
console.log(userName); // "Bartek"

Dlaczego unikamy var?

  • Hoisting - zmienna istnieje przed deklaracją
  • Brak block scope - wycieka z bloków {}
  • Można przypadkowo nadpisać zmienną
var to spuścizna starego JS (przed ES6/2015). W nowym kodzie zawsze używamy let/const. Warto wiedzieć, że istnieje - spotkamy go w starszym kodzie. Nie trzeba tłumaczyć hoistingu szczegółowo - wystarczy powiedzieć że var jest nieprzewidywalny.

Problem z var - hoisting


// Z var - dziala, ale wartosc to undefined
console.log(product); // undefined (!)
var product = "Krem";

// Z let - blad (tak powinno byc!)
console.log(item); // ReferenceError!
let item = "Mydlo";

Zasada

let i const chronią nas przed użyciem zmiennej przed jej zadeklarowaniem.

Hoisting z var sprawia, że zmienna istnieje od początku funkcji, ale ma wartość undefined. let i const rzucają błąd - co jest lepsze, bo od razu widzimy problem.

let vs const vs var

Cecha let const var
Zmiana wartości Tak Nie Tak
Block scope Tak Tak Nie
Hoisting TDZ TDZ undefined
Re-deklaracja Nie Nie Tak

Złota zasada

Używaj const domyślnie. let gdy musisz zmienić wartość. var - nigdy.

TDZ = Temporal Dead Zone - zmienna istnieje, ale nie można jej użyć przed deklaracją. W praktyce: const domyślnie, let jeśli wartość się zmienia, var nigdy.

Nazewnictwo zmiennych

Zasady

  • Zaczyna się od litery, _ lub $
  • Nie może zaczynać się od cyfry
  • Nie może być słowem kluczowym
  • Wielkość liter ma znaczenie

Konwencja: camelCase


// Dobrze
let productName = "Szampon";
let totalPrice = 49.99;
let isAvailable = true;

// Zle
let product_name = "Szampon";
let TOTALPRICE = 49.99;
let x = true;
camelCase to standard w JavaScript. Pierwsza litera mała, kolejne słowa z wielkiej. Nazwy powinny być opisowe - productName zamiast x czy p. W testach spotkamy też UPPER_CASE dla stałych konfiguracyjnych.

// Zle - niejasne, skrotowe

Wskazówka

Czytając nazwę zmiennej, powinieneś od razu wiedzieć co przechowuje.

Dobre nazewnictwo to połowa sukcesu. Kod czyta się wielokrotnie częściej niż pisze. Boolean zmienne zaczynamy od is, has, can, should.

To najdłuższy blok. Będziemy omawiać każdy typ osobno z przykładami. Przerwa po tym bloku.

Typy prymitywne w JavaScript

Typ Przykład Opis
string "tekst" Łańcuch znaków
number 42, 3.14 Liczba
boolean true, false Wartość logiczna
undefined undefined Brak przypisanej wartości
null null Celowy brak wartości

Skupiamy się na pierwszych 5 typach. Symbol i BigInt to typy zaawansowane - nie będziemy ich omawiać. Każdy typ będziemy omawiać szczegółowo na kolejnych slajdach.

String - łańcuch znaków


// Trzy sposoby tworzenia stringow
const single = 'Witaj';           // apostrofy
const double = "Rossmann";        // cudzyslowy
const backtick = `Sklep Rossmann`; // backticki

// Lacznie stringow (konkatenacja)
const greeting = "Witaj w " + "Rossmann";
console.log(greeting); // "Witaj w Rossmann"

// Dlugosc stringa
const brand = "Rossmann";
console.log(brand.length); // 8
Trzy sposoby tworzenia stringów. Apostrofy i cudzysłowy są równoważne. Backticki mają specjalną moc - interpolacja (będziemy omawiać za chwilę). length to właściwość, nie metoda - bez nawiasów!

Metody stringów


const product = "Szampon Rossmann Premium";

// Zmiana wielkosci liter
console.log(product.toUpperCase());
// "SZAMPON ROSSMANN PREMIUM"

console.log(product.toLowerCase());
// "szampon rossmann premium"

// Sprawdzanie zawartosci
console.log(product.includes("Rossmann")); // true
console.log(product.includes("Nivea"));    // false

// Wycinanie fragmentu
console.log(product.slice(0, 7));   // "Szampon"
console.log(product.slice(8));      // "Rossmann Premium"
Metody stringów są bardzo przydatne w testach - sprawdzanie tekstów na stronie, porównywanie, wycinanie fragmentów. includes() używamy np. w asercjach. slice(start, end) - end jest opcjonalny.

Więcej metod stringów


const email = "  anna@rossmann.pl  ";

// Usuwanie bialych znakow
console.log(email.trim()); // "anna@rossmann.pl"

// Zamiana fragmentow
const url = "https://rossmann.pl/products";
console.log(url.replace("products", "offers"));
// "https://rossmann.pl/offers"

// Sprawdzanie poczatku/konca
console.log(url.startsWith("https")); // true
console.log(url.endsWith(".pl/products")); // true

// Indeks znaku
console.log("Rossmann".indexOf("s")); // 2
trim() jest bardzo przydatny - formularze często mają dodatkowe spacje. replace() zamienia tylko pierwsze wystąpienie - replaceAll() zamienia wszystkie. Metody stringów nie zmieniają oryginalnego stringa - zwracają nowy!

Template literals - backticki


const productName = "Szampon Rossmann";
const price = 12.99;
const quantity = 3;

// Stary sposob (konkatenacja)
const oldWay = "Produkt: " + productName +
               ", cena: " + price + " PLN";

// Nowy sposob (template literal)
const newWay = `Produkt: ${productName}, cena: ${price} PLN`;

// Mozna wstawiac wyrazenia!
const total = `Razem: ${price * quantity} PLN`;
console.log(total); // "Razem: 38.97 PLN"

Wskazówka

Używaj ${} zamiast łączenia stringów przez +. Kod jest czytelniejszy.

Template literals to jeden z najważniejszych feature'ów ES6. Backtick to klawisz pod Esc (na klawiaturach PL może być inny skrót). W ${} można wstawić dowolne wyrażenie JS.

Template literals - wiele linii


// Stary sposob - trudne do czytania
const oldMessage = "Witaj!\n" +
                   "Twoje zamowienie:\n" +
                   "Produkt: Szampon\n" +
                   "Cena: 12.99 PLN";

// Template literal - naturalne linie
const newMessage = `Witaj!
Twoje zamowienie:
Produkt: Szampon
Cena: 12.99 PLN`;

console.log(newMessage);
Template literals zachowują znaki nowej linii - nie trzeba używać \n. Świetne do tworzenia dłuższych tekstów, np. treści emaili w testach.

Number - liczby


// Liczby calkowite (integer)
const quantity = 5;
const year = 2025;

// Liczby zmiennoprzecinkowe (float)
const price = 29.99;
const taxRate = 0.23;

// Obliczenia
const total = price * quantity;
const tax = total * taxRate;
console.log(`Razem: ${total} PLN`);     // 149.95
console.log(`VAT: ${tax.toFixed(2)} PLN`); // 34.49

// Specjalne wartosci
console.log(1 / 0);        // Infinity
console.log("abc" * 2);    // NaN (Not a Number)
W JS jest tylko jeden typ liczbowy - number. Obejmuje całkowite i zmiennoprzecinkowe. toFixed(2) zaokrągla do 2 miejsc po przecinku - zwraca stringa! NaN pojawia się gdy próbujemy zrobić matematykę na nie-liczbach.

NaN i Infinity


// NaN - Not a Number
const result = "hello" * 5;
console.log(result);       // NaN
console.log(typeof result); // "number" (!)

// Sprawdzanie czy cos jest NaN
console.log(isNaN(result));          // true
console.log(Number.isNaN(result));   // true (lepszy sposob)

// NaN nie jest rowne niczemu, nawet sobie!
console.log(NaN === NaN); // false (!)

// Infinity
console.log(1 / 0);    // Infinity
console.log(-1 / 0);   // -Infinity

Pułapka!

typeof NaN zwraca "number" - to jeden z dziwactw JS.

NaN to częste źródło błędów. NaN !== NaN to jeden z najbardziej zaskakujących faktów o JS. Zawsze używaj Number.isNaN() zamiast porównania z NaN.

Obiekt Math


// Zaokraglanie
console.log(Math.floor(4.7));   // 4 (w dol)
console.log(Math.ceil(4.2));    // 5 (w gore)
console.log(Math.round(4.5));   // 5 (najblizsza)

// Losowa liczba
console.log(Math.random());     // 0.0 - 0.999...

// Losowa calkowita 1-100
const randomNum = Math.floor(Math.random() * 100) + 1;
console.log(randomNum);

// Inne przydatne
console.log(Math.max(10, 20, 5));   // 20
console.log(Math.min(10, 20, 5));   // 5
console.log(Math.abs(-15));          // 15
Math to wbudowany obiekt z metodami matematycznymi. Math.random() jest przydatne w testach - losowe dane testowe. Math.floor() często łączymy z Math.random() żeby dostać losową liczbę całkowitą.

Boolean - wartość logiczna


const isLoggedIn = true;
const isAdmin = false;

// Boolean z porownania
const price = 29.99;
const isExpensive = price > 50;
console.log(isExpensive); // false

const age = 25;
const isAdult = age >= 18;
console.log(isAdult); // true

// W testach
const isVisible = true;
const isEnabled = false;
const hasError = false;
Boolean to typ z dwoma wartościami: true/false. Kluczowy w warunkach i testach. W testach często sprawdzamy: isVisible, isEnabled, hasError. Nazwy boolean zmiennych zaczynamy od is/has/can/should.

null vs undefined

undefined null
Znaczenie Brak wartości (domyślne) Celowy brak wartości
Kto ustawia JavaScript automatycznie Programista świadomie
typeof "undefined" "object" (!) 

let product;
console.log(product);  // undefined (JS ustawil)

let discount = null;
console.log(discount); // null (my ustawilismy)

// Porownanie
console.log(null == undefined);  // true (!)
console.log(null === undefined); // false
undefined - JS mówi "nie wiem, nie przypisano wartości". null - programista mówi "celowo nie ma tu wartości". typeof null === "object" to historyczny błąd w JS, który został na zawsze.

Operator typeof


console.log(typeof "Rossmann");   // "string"
console.log(typeof 42);           // "number"
console.log(typeof true);         // "boolean"
console.log(typeof undefined);    // "undefined"
console.log(typeof null);         // "object" (blad!)
console.log(typeof NaN);          // "number" (!)
console.log(typeof [1, 2, 3]);    // "object"
console.log(typeof {});           // "object"

Pułapki typeof

  • typeof null zwraca "object" - to znany błąd JS
  • typeof NaN zwraca "number" - mimo że to "Not a Number"
typeof jest przydatny do sprawdzania typów w runtime. Dwie pułapki warto zapamiętać: typeof null i typeof NaN. W testach często używamy typeof do walidacji danych z API.

Jawna konwersja typów


// Na string
const num = 42;
console.log(String(num));    // "42"
console.log(num.toString()); // "42"

// Na number
const str = "123";
console.log(Number(str));     // 123
console.log(Number("abc"));   // NaN
console.log(parseInt("50px")); // 50
console.log(parseFloat("3.14abc")); // 3.14

// Na boolean
console.log(Boolean(1));     // true
console.log(Boolean(0));     // false
console.log(Boolean(""));    // false
console.log(Boolean("abc")); // true
Jawna konwersja - my świadomie konwertujemy typ. Number() jest bardziej rygorystyczny niż parseInt() - Number("50px") daje NaN. parseInt/parseFloat parsują ile się da i ignorują resztę.

Niejawna konwersja (coercion)


// String + number = string (konkatenacja!)
console.log("5" + 3);     // "53" (!)
console.log("5" + true);  // "5true" (!)

// Z innymi operatorami - konwersja na number
console.log("5" - 3);     // 2
console.log("5" * 2);     // 10
console.log("5" / 2);     // 2.5

// Niejawna konwersja na boolean
if ("hello") { /* wykona sie */ }
if (0)       { /* NIE wykona sie */ }

Uwaga!

Operator + z stringiem zawsze łączy stringi, nie dodaje!

Niejawna konwersja to jedno z największych źródeł błędów w JS. + jest przeciążony: dodawanie liczb LUB łączenie stringów. Dlatego ważne jest używanie === zamiast == (o tym za chwilę).

Operatory

Wykonywanie operacji na danych

Operatory to symbole które mówią JS co zrobić z wartościami. Najważniejszy temat w tym bloku: == vs ===.

Operatory arytmetyczne


const a = 10;
const b = 3;

console.log(a + b);   // 13  - dodawanie
console.log(a - b);   // 7   - odejmowanie
console.log(a * b);   // 30  - mnożenie
console.log(a / b);   // 3.333... - dzielenie
console.log(a % b);   // 1   - reszta z dzielenia (modulo)
console.log(a ** b);  // 1000 - potęgowanie (10^3)

// Praktyczny przyklad
const totalItems = 17;
const itemsPerPage = 5;
const fullPages = Math.floor(totalItems / itemsPerPage); // 3
const remainingItems = totalItems % itemsPerPage; // 2
Operatory arytmetyczne działają jak w matematyce. Modulo (%) przydaje się np. do sprawdzania parzystości, paginacji. Potęgowanie (**) to nowszy operator.

Operatory przypisania


let score = 100;

score = score + 10;  // 110 - pelna forma
score += 10;         // 120 - skrocona forma (to samo!)

score -= 20;         // 100
score *= 2;          // 200
score /= 4;          // 50

// Inkrementacja / Dekrementacja
let count = 0;
count++;    // 1 (count = count + 1)
count++;    // 2
count--;    // 1 (count = count - 1)

Skróty

x += 5 to to samo co x = x + 5

Operatory przypisania to skróty - robią to samo co pełna forma, ale krócej. ++ i -- są bardzo częste w pętlach (będziemy omawiać w kolejnych dniach).

== vs === (najważniejszy slajd!)


// == (luźne porownanie) - konwertuje typy!
console.log(5 == "5");     // true (!)
console.log(0 == false);   // true (!)
console.log("" == false);  // true (!)
console.log(null == undefined); // true (!)

// === (scisle porownanie) - bez konwersji
console.log(5 === "5");     // false (rozne typy)
console.log(0 === false);   // false
console.log("" === false);  // false
console.log(null === undefined); // false

ZASADA NUMER 1

Zawsze używaj === i !==. Nigdy == i !=.

TO JEST NAJWAŻNIEJSZY SLAJD O OPERATORACH. Powiedzieć to wyraźnie. == konwertuje typy przed porównaniem - to prowadzi do nieoczekiwanych wyników. === porównuje wartość I typ - bezpieczniejsze, przewidywalne. Każdy linter (ESLint) domyślnie zakazuje == na rzecz ===.

Operatory porównania


const price = 29.99;
const budget = 50;

console.log(price > budget);   // false
console.log(price < budget);   // true
console.log(price >= 29.99);   // true
console.log(price <= 30);      // true

// Porownywanie stringow (alfabetycznie)
console.log("apple" < "banana"); // true
console.log("a" < "b");         // true

// Nie-rownosc
console.log(5 !== "5");  // true (rozne typy - DOBRZE)
console.log(5 != "5");   // false (konwersja - ZLE)
Operatory > < >= <= działają intuicyjnie z liczbami. Stringi porównywane są leksykograficznie (alfabetycznie wg kodów Unicode). Przypomnienie: !== zamiast !=, tak jak === zamiast ==.

Operatory logiczne


const isLoggedIn = true;
const isAdmin = false;
const hasPermission = true;

// AND (&&) - oba musza byc true
console.log(isLoggedIn && isAdmin);       // false
console.log(isLoggedIn && hasPermission); // true

// OR (||) - wystarczy jedno true
console.log(isLoggedIn || isAdmin);  // true
console.log(false || false);          // false

// NOT (!) - negacja
console.log(!isLoggedIn);  // false
console.log(!isAdmin);     // true
console.log(!!0);          // false (podwojna negacja = Boolean())
Operatory logiczne to podstawa warunków. && - oba muszą być prawdziwe. || - wystarczy jedno prawdziwe. ! - odwraca wartość. !! to trick na konwersję do boolean (to samo co Boolean()).

Operatory logiczne - praktyczne przykłady


const age = 25;
const hasID = true;
const isMember = false;

// Sprawdzenie wielu warunkow
const canBuyAlcohol = age >= 18 && hasID;
console.log(canBuyAlcohol); // true

// Rabat dla czlonkow LUB starszych niz 65
const getsDiscount = isMember || age >= 65;
console.log(getsDiscount); // false

// Wartosci domyslne z ||
const userName = "" || "Gość";
console.log(userName); // "Gosc" (pusty string jest falsy)

// Nullish coalescing (??)
const count = 0 ?? 10;
console.log(count); // 0 (?? tylko null/undefined)
|| jest często używany do wartości domyślnych, ale ma pułapkę z 0 i "". ?? (nullish coalescing) - nowszy operator, reaguje tylko na null/undefined. Przykłady z testowania: sprawdzanie czy element jest widoczny I klikalny.

Operator warunkowy (ternary)


// Skladnia: warunek ? jesliTak : jesliNie

const age = 20;
const status = age >= 18 ? "dorosly" : "niepelnoletni";
console.log(status); // "dorosly"

// Praktyczne przyklady
const price = 49.99;
const shipping = price > 100 ? 0 : 9.99;
console.log(`Dostawa: ${shipping} PLN`); // "Dostawa: 9.99 PLN"

const score = 85;
const grade = score >= 90 ? "A" :
              score >= 80 ? "B" :
              score >= 70 ? "C" : "F";
console.log(grade); // "B"

Kiedy używać?

Dla prostych warunków. Zagnieżdżone ternary są trudne do czytania.

Ternary to skrócony if/else w jednej linii. Świetny do prostych warunków. Zagnieżdżone ternary (jak grade) są kontrowersyjne - lepiej użyć if/else. Będziemy omawiać if/else szczegółowo w dniu 2.

Priorytet operatorów


// Tak jak w matematyce - mnozenie przed dodawaniem
console.log(2 + 3 * 4);   // 14 (nie 20!)
console.log((2 + 3) * 4); // 20 (nawiasy zmieniaja priorytet)

// Priorytet (od najwyzszego):
// 1. () - nawiasy
// 2. ** - potegowanie
// 3. *, /, % - mnozenie, dzielenie, modulo
// 4. +, - - dodawanie, odejmowanie
// 5. <, >, <=, >= - porownania
// 6. ===, !== - rownosc
// 7. && - logiczne AND
// 8. || - logiczne OR

Prosta zasada

Kiedy nie jesteś pewien - użyj nawiasów ()!

Nie trzeba uczyć się całej tabeli na pamięć. Najważniejsze: nawiasy mają najwyższy priorytet. W wątpliwości - nawiasy. Czytelność kodu jest ważniejsza niż zwięzłość.
