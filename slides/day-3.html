01

Tablice

Uporządkowane kolekcje danych

Przechodzimy do tablic - jednej z najważniejszych struktur danych w JavaScript.

Czym jest tablica?

Tablica to uporządkowana lista elementów

Analogia

Lista zakupów w Rossmann:

  1. Szampon
  2. Pasta do zębów
  3. Krem nawilżający
  4. Dezodorant

W JavaScript


let shoppingList = [
  "Szampon",
  "Pasta do zębów",
  "Krem nawilżający",
  "Dezodorant"
];
Użyj prostej analogii - lista zakupów. Każdy element ma swoją pozycję (indeks). Tablica może przechowywać dowolne typy danych.

Tworzenie tablic


// Literał tablicy - preferowany sposób
let products = ["Szampon", "Krem", "Pasta"];

// Pusta tablica
let emptyList = [];

// Tablica z różnymi typami
let mixed = ["tekst", 42, true, null];

// Konstruktor Array (rzadko używany)
let numbers = new Array(1, 2, 3);

Wskazówka

Zawsze używaj zapisu [] zamiast new Array(). Jest prostszy i bezpieczniejszy.

Pokaż, że tablice mogą zawierać różne typy - ale w praktyce najczęściej trzymamy elementy tego samego typu. Ostrzeż przed new Array(3) - tworzy pustą tablicę o długości 3, nie tablicę [3].

Indeksowanie - dostęp do elementów


let products = ["Szampon", "Krem", "Pasta", "Dezodorant"];
//  indeks:       0          1       2         3

console.log(products[0]);  // "Szampon"
console.log(products[2]);  // "Pasta"
console.log(products[4]);  // undefined (nie istnieje!)

Uwaga

Indeksowanie zaczyna się od zera, nie od jedynki! Pierwszy element to [0], drugi to [1] itd.

To klasyczny błąd off-by-one. Podkreśl, że próba dostępu do nieistniejącego indeksu zwraca undefined, a nie powoduje błąd.

.length - długość tablicy


let products = ["Szampon", "Krem", "Pasta"];

console.log(products.length);  // 3

// Ostatni element
console.log(products[products.length - 1]);  // "Pasta"

// Pusta tablica
let empty = [];
console.log(empty.length);  // 0

// Sprawdzanie czy tablica jest pusta
if (products.length > 0) {
  console.log("Lista nie jest pusta");
}
Pokaż pattern dostępu do ostatniego elementu: array[array.length - 1]. Przy okazji przypomnij, że length to właściwość, nie metoda - bez nawiasów.

push() i pop()

Dodawanie i usuwanie z końca tablicy


let cart = ["Szampon", "Krem"];

// push() - dodaje na koniec
cart.push("Pasta");
console.log(cart);  // ["Szampon", "Krem", "Pasta"]

cart.push("Mydło", "Żel");
console.log(cart);  // ["Szampon", "Krem", "Pasta", "Mydło", "Żel"]

// pop() - usuwa z końca (i zwraca usunięty element)
let lastItem = cart.pop();
console.log(lastItem);  // "Żel"
console.log(cart);       // ["Szampon", "Krem", "Pasta", "Mydło"]
push i pop to najczęściej używane metody. Podkreśl, że push może przyjąć wiele argumentów naraz, a pop zwraca usunięty element.

shift() i unshift()

Dodawanie i usuwanie z początku tablicy


let queue = ["Anna", "Bartek", "Celina"];

// unshift() - dodaje na początek
queue.unshift("Dorota");
console.log(queue);  // ["Dorota", "Anna", "Bartek", "Celina"]

// shift() - usuwa z początku (i zwraca usunięty element)
let firstPerson = queue.shift();
console.log(firstPerson);  // "Dorota"
console.log(queue);         // ["Anna", "Bartek", "Celina"]

Analogia

push/pop - stos talerzy (ostatni kładziony, pierwszy zdejmowany)
unshift/shift - kolejka w sklepie (nowi na koniec, obsługa z początku)

Nazwy shift/unshift mogą być mylące. Użyj analogii - push/pop to stos, shift/unshift to kolejka. shift i unshift są wolniejsze, bo przeindeksowują wszystkie elementy.

splice() - operacje w środku tablicy


let products = ["Szampon", "Krem", "Pasta", "Mydło", "Żel"];

// Usuwanie: splice(start, ilość)
products.splice(1, 2);
console.log(products);  // ["Szampon", "Mydło", "Żel"]

// Dodawanie: splice(start, 0, ...elementy)
products.splice(1, 0, "Balsam", "Odżywka");
console.log(products);  // ["Szampon", "Balsam", "Odżywka", "Mydło", "Żel"]

// Zamiana: splice(start, ilość, ...nowe)
products.splice(0, 1, "Szampon XL");
console.log(products);  // ["Szampon XL", "Balsam", "Odżywka", "Mydło", "Żel"]
splice to szwajcarski scyzoryk tablic - potrafi usuwać, dodawać i zamieniać. Podkreśl, że splice MUTUJE oryginalną tablicę. Pierwszy argument to indeks startowy, drugi to liczba elementów do usunięcia, kolejne to elementy do dodania.

includes() - czy element istnieje?


let products = ["Szampon", "Krem", "Pasta"];

console.log(products.includes("Krem"));     // true
console.log(products.includes("Dezodorant")); // false

// Użycie w warunku
if (products.includes("Szampon")) {
  console.log("Szampon jest na liście!");
}

// Uwaga na wielkość liter!
console.log(products.includes("szampon")); // false

Uwaga

includes() porównuje wartości ściśle (===), więc rozróżnia wielkość liter.

includes jest bardzo przydatne w testach - np. sprawdzanie, czy dany element znajduje się w odpowiedzi z API. Podkreśl case-sensitivity.

indexOf() - znajdowanie indeksu


let products = ["Szampon", "Krem", "Pasta", "Krem"];

console.log(products.indexOf("Krem"));       // 1 (pierwsze wystąpienie)
console.log(products.indexOf("Dezodorant")); // -1 (nie znaleziono!)

// Sprawdzanie czy element istnieje (stary sposób)
if (products.indexOf("Pasta") !== -1) {
  console.log("Pasta znaleziona na indeksie:", products.indexOf("Pasta"));
}

// lastIndexOf - szuka od końca
console.log(products.lastIndexOf("Krem"));  // 3

Wskazówka

Jeśli potrzebujesz tylko sprawdzić obecność - użyj includes(). indexOf() przydaje się, gdy potrzebujesz konkretnej pozycji.

indexOf zwraca -1 gdy element nie istnieje. Przed includes() to był jedyny sposób na sprawdzenie obecności elementu. Pokaż oba podejścia.

slice() - wycinanie fragmentu


let products = ["Szampon", "Krem", "Pasta", "Mydło", "Żel"];

// slice(start, end) - end NIE jest uwzględniony
let subset = products.slice(1, 3);
console.log(subset);    // ["Krem", "Pasta"]
console.log(products);  // ["Szampon", "Krem", "Pasta", "Mydło", "Żel"]
                        // oryginał niezmieniony!

// Tylko start - do końca
console.log(products.slice(2));  // ["Pasta", "Mydło", "Żel"]

// Ujemne indeksy - od końca
console.log(products.slice(-2));  // ["Mydło", "Żel"]

// Kopia tablicy
let copy = products.slice();
Kluczowa różnica: slice NIE mutuje oryginalnej tablicy (w przeciwieństwie do splice). Podkreśl, że end jest exclusive - element na pozycji end nie jest uwzględniony.

Mutacja vs tworzenie nowej tablicy

Mutujące (zmieniają oryginał)

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()

Niemutujące (tworzą nową)

  • slice()
  • map()
  • filter()
  • concat()
  • flat()
  • toSorted()
  • toReversed()

Uwaga

Mutacja to częsty źródło błędów! Zawsze sprawdź, czy metoda zmienia oryginał.

To bardzo ważna koncepcja. W testach automatycznych często pracujemy na danych, które nie powinny być zmieniane między testami. Pokaż, jak łatwo o niespodziewany efekt uboczny.

Przykład: Lista produktów


let products = ["Szampon", "Krem", "Pasta"];

// Dodaj nowy produkt
products.push("Mydło");

// Sprawdź czy produkt jest na liście
if (products.includes("Krem")) {
  console.log("Krem jest dostępny!");
}

// Usuń produkt o podanym indeksie
let indexToRemove = products.indexOf("Pasta");
if (indexToRemove !== -1) {
  products.splice(indexToRemove, 1);
}

console.log(products);  // ["Szampon", "Krem", "Mydło"]
console.log(`Produktów: ${products.length}`);  // Produktów: 3
Pokaż realny scenariusz - zarządzanie listą produktów. Zwróć uwagę na pattern: najpierw indexOf, potem sprawdzenie czy !== -1, a potem splice.

Przykład: Lista przypadków testowych


let testCases = [
  "Login z poprawnymi danymi",
  "Login z błędnym hasłem",
  "Rejestracja nowego użytkownika",
  "Reset hasła"
];

console.log(`Liczba testów: ${testCases.length}`);

// Dodaj nowy test
testCases.push("Wylogowanie użytkownika");

// Pobierz pierwsze 3 testy do uruchomienia
let firstBatch = testCases.slice(0, 3);
console.log("Pierwsza partia:", firstBatch);

// Sprawdź czy test istnieje
let searchTerm = "Reset hasła";
console.log(`"${searchTerm}" - istnieje: ${testCases.includes(searchTerm)}`);
Przykład bliski testerom - zarządzanie listą przypadków testowych. Pokaż, jak metody tablicowe mogą być przydatne w codziennej pracy testera.

Ćwiczenie 3.1 - Podstawowe operacje na tablicach

  2. Wyświetl trzeci element z listy
  3. Dodaj 2 nowe produkty na koniec
  4. Usuń pierwszy element z listy
  5. Sprawdź, czy "Szampon" jest na liście
  6. Znajdź indeks wybranego produktu

  8. Wyświetl długość obu tablic
Daj uczestnikom 15-20 minut na to ćwiczenie. Chodź po sali i pomagaj. Najczęstszy problem: pomylenie slice i splice.

Ćwiczenie: Koszyk zakupowy

Ćwiczenie 3.2 - Zarządzanie koszykiem

  1. Utwórz pustą tablicę cart
  2. Napisz funkcję addToCart(product) - dodaje produkt, jeśli go jeszcze nie ma
  3. Napisz funkcję removeFromCart(product) - usuwa produkt z koszyka
  4. Napisz funkcję showCart() - wyświetla zawartość i liczbę produktów
  5. Przetestuj dodając, usuwając i wyświetlając produkty
Ćwiczenie łączy tablice z funkcjami z dnia 2. Podpowiedź: includes() do sprawdzenia duplikatów, indexOf() + splice() do usuwania.

Rozwiązanie: Koszyk zakupowy


let cart = [];

function addToCart(product) {
  if (cart.includes(product)) {
    console.log(`"${product}" jest już w koszyku!`);
  } else {
    cart.push(product);
    console.log(`Dodano "${product}" do koszyka.`);
  }
}

function removeFromCart(product) {
  let index = cart.indexOf(product);
  if (index !== -1) {
    cart.splice(index, 1);
    console.log(`Usunięto "${product}" z koszyka.`);
  } else {
    console.log(`"${product}" nie ma w koszyku!`);
  }
}

function showCart() {
  console.log(`Koszyk (${cart.length}):`, cart);
}
Omów rozwiązanie krok po kroku. Podkreśl pattern indexOf + splice do usuwania po wartości.
02

Iteracja po tablicach

forEach, map, filter i inne metody wyższego rzędu

Teraz przejdziemy do potężnych metod iteracyjnych. To one sprawiają, że JavaScript jest tak wygodny do pracy z danymi.

forEach() - iteracja po elementach


let products = ["Szampon", "Krem", "Pasta"];

// forEach przyjmuje funkcję (callback)
products.forEach(function(product) {
  console.log("Produkt:", product);
});

// Z arrow function (krótsza forma)
products.forEach((product, index) => {
  console.log(`${index + 1}. ${product}`);
});
1. Szampon
2. Krem
3. Pasta
forEach to najprostszy sposób na przejście po tablicy. Podkreśl, że callback dostaje element i opcjonalnie indeks. forEach nic nie zwraca (undefined).

map() - transformacja elementów


let prices = [10, 25, 50, 15];

// map() tworzy NOWĄ tablicę z przetworzonymi elementami
let discountedPrices = prices.map(price => price * 0.9);
console.log(discountedPrices);  // [9, 22.5, 45, 13.5]
console.log(prices);            // [10, 25, 50, 15] - oryginał niezmieniony!

// Dodanie waluty
let formatted = prices.map(price => `${price} zł`);
console.log(formatted);  // ["10 zł", "25 zł", "50 zł", "15 zł"]

// Wyciągnięcie właściwości
let names = ["anna", "bartek", "celina"];
let upper = names.map(name => name.toUpperCase());
console.log(upper);  // ["ANNA", "BARTEK", "CELINA"]
map to jedna z najważniejszych metod. Kluczowe: tworzy NOWĄ tablicę tej samej długości. Każdy element jest przetransformowany przez funkcję callback.

map() vs forEach()

forEach()

  • Nie zwraca nowej tablicy
  • Służy do wykonania akcji
  • np. wyświetlanie, logowanie

products.forEach(p => {
  console.log(p);
});
// zwraca: undefined

map()

  • Zwraca NOWĄ tablicę
  • Służy do transformacji
  • np. zmiana formatu danych

let upper = products.map(p => {
  return p.toUpperCase();
});
// zwraca: ["SZAMPON", ...]
To kluczowe rozróżnienie. Jeśli potrzebujesz nowej tablicy - map. Jeśli chcesz tylko coś zrobić z każdym elementem (np. wyświetlić) - forEach.

filter() - filtrowanie elementów


let prices = [5, 12, 35, 8, 50, 22, 3];

// Filtruj produkty droższe niż 10 zł
let expensive = prices.filter(price => price > 10);
console.log(expensive);  // [12, 35, 50, 22]

// Filtruj parzyste
let even = prices.filter(price => price % 2 === 0);
console.log(even);  // [12, 8, 50, 22]

// Filtruj po tekście
let products = ["Szampon Head", "Krem Nivea", "Szampon Dove", "Pasta Colgate"];
let shampoos = products.filter(p => p.startsWith("Szampon"));
console.log(shampoos);  // ["Szampon Head", "Szampon Dove"]

Zapamiętaj

filter() zwraca nową tablicę z elementami, dla których callback zwrócił true.

filter jest niezwykle przydatny w testach - np. filtrowanie błędów z logów, filtrowanie odpowiedzi API po statusie. Podkreśl, że tablica wynikowa może być krótsza lub pusta.

find() - znajdowanie pierwszego pasującego


let prices = [5, 12, 35, 8, 50];

// Znajdź pierwszy element większy niż 10
let found = prices.find(price => price > 10);
console.log(found);  // 12 (nie tablica, ale sam element!)

// Jeśli nie znaleziono
let notFound = prices.find(price => price > 100);
console.log(notFound);  // undefined

// Szukanie w tablicy stringów
let products = ["Szampon", "Krem nawilżający", "Pasta", "Krem ochronny"];
let cream = products.find(p => p.includes("Krem"));
console.log(cream);  // "Krem nawilżający" (pierwszy pasujący)

find() vs filter()

find() zwraca pierwszy pasujący element (lub undefined). filter() zwraca wszystkie pasujące (jako tablicę).

find jest przydatny, gdy szukamy jednego konkretnego elementu. Zwraca sam element, nie tablicę. Jeśli nie znajdzie - zwraca undefined.

some() i every()


let prices = [5, 12, 35, 8, 50];

// some() - czy PRZYNAJMNIEJ JEDEN spełnia warunek?
console.log(prices.some(p => p > 40));   // true (50 > 40)
console.log(prices.some(p => p > 100));  // false

// every() - czy WSZYSTKIE spełniają warunek?
console.log(prices.every(p => p > 0));   // true (wszystkie dodatnie)
console.log(prices.every(p => p > 10));  // false (5 i 8 nie spełniają)

some()

"Czy jest choć jeden taki, że...?"

every()

"Czy wszystkie są takie, że...?"

some i every zwracają boolean - bardzo przydatne w asercjach testowych. Analogia: some = operator OR (przynajmniej jeden), every = operator AND (każdy).

some() / every() w testach


let testResults = [
  { name: "Login test", passed: true },
  { name: "Cart test", passed: false },
  { name: "Search test", passed: true }
];

// Czy wszystkie testy przeszły?
let allPassed = testResults.every(test => test.passed);
console.log(`Wszystkie testy OK: ${allPassed}`);  // false

// Czy jakikolwiek test nie przeszedł?
let anyFailed = testResults.some(test => !test.passed);
console.log(`Są błędne testy: ${anyFailed}`);  // true

// Walidacja danych - czy wszystkie ceny są dodatnie?
let prices = [10, 25, -5, 30];
let allPositive = prices.every(p => p > 0);
console.log(`Poprawne ceny: ${allPositive}`);  // false
Pokaż kontekst testowy. some i every są idealne do sprawdzania warunków na zbiorze danych - np. walidacja danych z API.

reduce() - redukcja do jednej wartości


let prices = [10, 25, 50, 15];

// Sumowanie - najczęstsze zastosowanie
let total = prices.reduce((sum, price) => {
  return sum + price;
}, 0);  // 0 to wartość początkowa
console.log(total);  // 100

// Krótsza forma
let total2 = prices.reduce((sum, price) => sum + price, 0);

// Znajdowanie maksimum
let max = prices.reduce((biggest, price) => {
  return price > biggest ? price : biggest;
}, 0);
console.log(max);  // 50
reduce to najsilniejsza metoda iteracyjna - ale też najtrudniejsza do zrozumienia. Na tym etapie skup się na sumowaniu i zliczaniu. Podkreśl rolę wartości początkowej (akumulatora).

reduce() - zliczanie elementów


let votes = ["tak", "nie", "tak", "tak", "nie", "tak"];

let result = votes.reduce((counts, vote) => {
  counts[vote] = (counts[vote] || 0) + 1;
  return counts;
}, {});

console.log(result);  // { tak: 4, nie: 2 }

// Zliczanie produktów po kategorii
let categories = ["kosmetyki", "higiena", "kosmetyki", "higiena", "jedzenie"];

let categoryCounts = categories.reduce((acc, category) => {
  acc[category] = (acc[category] || 0) + 1;
  return acc;
}, {});

console.log(categoryCounts);  // { kosmetyki: 2, higiena: 2, jedzenie: 1 }
Zliczanie to drugiej najczęstsze użycie reduce. Akumulator to obiekt, do którego dodajemy klucze. Jeśli to zbyt trudne - wróć do tego później. Nie wszyscy muszą to teraz opanować.

Chainowanie metod


let products = [
  { name: "Szampon", price: 15, category: "higiena" },
  { name: "Krem", price: 45, category: "kosmetyki" },
  { name: "Pasta", price: 8, category: "higiena" },
  { name: "Perfumy", price: 120, category: "kosmetyki" },
  { name: "Mydło", price: 5, category: "higiena" }
];

// Znajdź nazwy produktów z kategorii "higiena" tańszych niż 10 zł
let result = products
  .filter(p => p.category === "higiena")
  .filter(p => p.price < 10)
  .map(p => p.name);

console.log(result);  // ["Pasta", "Mydło"]

Wskazówka

Metody filter() i map() zwracają tablice, więc możemy wywoływać kolejne metody w łańcuchu (chain).

Chainowanie to potężna technika. Czyta się od góry do dołu: najpierw filtruj, potem transformuj. Każda metoda zwraca nową tablicę, na której można wywołać kolejną metodę.

Chainowanie - przykład z sumowaniem


let products = [
  { name: "Szampon", price: 15, inStock: true },
  { name: "Krem", price: 45, inStock: false },
  { name: "Pasta", price: 8, inStock: true },
  { name: "Perfumy", price: 120, inStock: true },
  { name: "Mydło", price: 5, inStock: true }
];

// Łączna cena dostępnych produktów
let totalInStock = products
  .filter(p => p.inStock)
  .map(p => p.price)
  .reduce((sum, price) => sum + price, 0);

console.log(`Wartość dostępnych: ${totalInStock} zł`);  // 148 zł

// Ile produktów jest dostępnych?
let inStockCount = products.filter(p => p.inStock).length;
console.log(`Dostępnych: ${inStockCount}`);  // 4
Pokaż, jak elegancko można łączyć filter + map + reduce. Przy okazji pokaż, że .length na wyniku filter to prosty sposób na zliczenie pasujących elementów.

Przykład: Analiza wyników testów


let tests = [
  { name: "TC-001", status: "passed", duration: 120 },
  { name: "TC-002", status: "failed", duration: 350 },
  { name: "TC-003", status: "passed", duration: 90 },
  { name: "TC-004", status: "skipped", duration: 0 },
  { name: "TC-005", status: "failed", duration: 500 }
];

let failedTests = tests.filter(t => t.status === "failed");
console.log("Niezdane:", failedTests.map(t => t.name));
// ["TC-002", "TC-005"]

let avgDuration = tests
  .filter(t => t.status !== "skipped")
  .map(t => t.duration)
  .reduce((sum, d) => sum + d, 0) / tests.filter(t => t.status !== "skipped").length;

console.log(`Średni czas: ${avgDuration}ms`);
Przykład bliski sercu testerów - analiza wyników testów. Pokaż, jak filter + map + reduce pozwalają łatwo wyciągać statystyki z danych.

Obiekty

Kolekcje par klucz-wartość

Przechodzimy do obiektów - drugiej kluczowej struktury danych w JavaScript. Obiekty to fundament JSON, a więc i pracy z API.

Czym jest obiekt?

Obiekt to zbiór par klucz-wartość

Analogia

Karta produktu w Rossmann:

  • Nazwa: Szampon Head & Shoulders
  • Cena: 24.99 zł
  • Kategoria: Higiena
  • Dostępny: Tak

W JavaScript


let product = {
  name: "Szampon H&S",
  price: 24.99,
  category: "Higiena",
  inStock: true
};
Obiekt to jak formularz lub karta - ma pola (klucze) i wartości. Tablica = lista elementów, obiekt = zbiór nazwanych właściwości.

Tworzenie obiektu


// Literał obiektowy - preferowany sposób
let user = {
  firstName: "Anna",
  lastName: "Kowalska",
  age: 30,
  isAdmin: false
};

// Pusty obiekt
let emptyObj = {};

// Zagnieżdżone wartości
let product = {
  name: "Krem Nivea",
  price: 29.99,
  tags: ["kosmetyki", "twarz", "nawilżenie"],  // tablica jako wartość
  manufacturer: {                                // obiekt jako wartość
    name: "Beiersdorf",
    country: "Niemcy"
  }
};
Pokaż, że wartości mogą być dowolnego typu - stringi, liczby, boolean, tablice, a nawet inne obiekty. To właśnie daje obiektom elastyczność.

Dostęp do właściwości


let user = { firstName: "Anna", lastName: "Kowalska", age: 30 };

// Dot notation - najczęściej używana
console.log(user.firstName);  // "Anna"
console.log(user.age);        // 30

// Bracket notation - z użyciem stringa
console.log(user["lastName"]);  // "Kowalska"

// Bracket notation - z dynamicznym kluczem
let field = "firstName";
console.log(user[field]);  // "Anna"

// Dodawanie nowych właściwości
user.email = "anna@example.com";
user["phone"] = "123-456-789";

// Usuwanie właściwości
delete user.phone;

Kiedy bracket notation?

Gdy klucz jest w zmiennej, zawiera spacje lub znaki specjalne, np. user["full-name"].

Dot notation jest prostsza i częstsza. Bracket notation potrzebna, gdy klucz pochodzi ze zmiennej (np. z danych dynamicznych) lub zawiera znaki specjalne.

Modyfikacja obiektu


let product = {
  name: "Szampon",
  price: 15.99,
  inStock: true
};

// Zmiana wartości
product.price = 12.99;
console.log(product.price);  // 12.99

// Dodanie nowej właściwości
product.discount = 20;

// Sprawdzenie czy klucz istnieje
console.log("name" in product);      // true
console.log("color" in product);     // false
console.log(product.color);          // undefined

// Usunięcie właściwości
delete product.discount;
console.log(product);
// { name: "Szampon", price: 12.99, inStock: true }
Obiekty są mutowalne - możemy dowolnie dodawać, zmieniać i usuwać właściwości. Operator "in" sprawdza istnienie klucza. Dostęp do nieistniejącego klucza zwraca undefined.

Zagnieżdżone obiekty


let order = {
  id: 12345,
  customer: {
    name: "Anna Kowalska",
    address: {
      street: "ul. Marszałkowska 10",
      city: "Warszawa",
      zip: "00-001"
    }
  },
  items: [
    { name: "Szampon", quantity: 2, price: 15.99 },
    { name: "Krem", quantity: 1, price: 45.00 }
  ]
};

// Dostęp do zagnieżdżonych właściwości
console.log(order.customer.name);            // "Anna Kowalska"
console.log(order.customer.address.city);    // "Warszawa"
console.log(order.items[0].name);            // "Szampon"
console.log(order.items.length);             // 2
Zagnieżdżone obiekty to codzienność w API - odpowiedzi JSON często mają głęboką strukturę. Pokaż, jak łączymy dot notation z indeksowaniem tablic.

Bezpieczny dostęp - ?.


let user = {
  name: "Anna",
  address: null
};

// Niebezpiecznie - błąd!
// console.log(user.address.city);  // TypeError!

// Bezpiecznie - optional chaining
console.log(user.address?.city);     // undefined (bez błędu)
console.log(user.phone?.number);     // undefined (bez błędu)

// Przydatne przy danych z API
let response = { data: { users: [{ name: "Anna" }] } };
let firstName = response.data?.users?.[0]?.name;
console.log(firstName);  // "Anna"

let missing = response.data?.products?.[0]?.name;
console.log(missing);  // undefined

Wskazówka

?. zatrzymuje się i zwraca undefined jeśli wartość po lewej to null lub undefined.

Optional chaining to ratunek przy pracy z nieprzewidywalnymi danymi z API. Bez niego musielibyśmy sprawdzać każdy poziom zagnieżdżenia osobno.

Metody w obiekcie


let cart = {
  items: [],

  addItem(product) {
    this.items.push(product);
    console.log(`Dodano: ${product}`);
  },

  removeItem(product) {
    let index = this.items.indexOf(product);
    if (index !== -1) {
      this.items.splice(index, 1);
      console.log(`Usunięto: ${product}`);
    }
  },

  getCount() {
    return this.items.length;
  }
};

cart.addItem("Szampon");
cart.addItem("Krem");
console.log(cart.getCount());  // 2
cart.removeItem("Szampon");
console.log(cart.items);       // ["Krem"]
Metoda to funkcja będąca wartością właściwości obiektu. this odnosi się do samego obiektu. Pokaż krótki zapis metod (bez ": function").

Object.keys(), values(), entries()


let product = {
  name: "Szampon",
  price: 15.99,
  category: "higiena",
  inStock: true
};

// Wszystkie klucze
console.log(Object.keys(product));
// ["name", "price", "category", "inStock"]

// Wszystkie wartości
console.log(Object.values(product));
// ["Szampon", 15.99, "higiena", true]

// Pary [klucz, wartość]
console.log(Object.entries(product));
// [["name","Szampon"], ["price",15.99], ["category","higiena"], ["inStock",true]]

// Liczba właściwości
console.log(Object.keys(product).length);  // 4
Te trzy metody zamieniają obiekt na tablicę - dzięki czemu możemy użyć map, filter itp. Object.entries zwraca tablicę par, co jest szczególnie przydatne.

Iterowanie po obiekcie


let scores = { anna: 85, bartek: 92, celina: 78, damian: 95 };

// for...in - iteruje po kluczach
for (let name in scores) {
  console.log(`${name}: ${scores[name]} pkt`);
}

// Object.entries() + forEach
Object.entries(scores).forEach(([name, score]) => {
  console.log(`${name}: ${score} pkt`);
});

// Object.entries() + filter + map
let topStudents = Object.entries(scores)
  .filter(([name, score]) => score >= 90)
  .map(([name, score]) => name);

console.log("Najlepsi:", topStudents);  // ["bartek", "damian"]
for...in to najprostszy sposób, ale Object.entries z destructuringiem jest czytelniejszy i pozwala na chainowanie z filter/map. Pokaż oba podejścia.

Destructuring & Spread

Wygodne wydobywanie i łączenie danych

Destructuring i spread to nowoczesne składnie ES6+, które znacznie upraszczają pracę z tablicami i obiektami.

Destructuring tablic


let colors = ["czerwony", "zielony", "niebieski"];

// Bez destructuringu
let first = colors[0];
let second = colors[1];

// Z destructuringiem
let [r, g, b] = colors;
console.log(r);  // "czerwony"
console.log(g);  // "zielony"
console.log(b);  // "niebieski"

// Pomijanie elementów
let [, , third] = colors;
console.log(third);  // "niebieski"

// Wartości domyślne
let [a, x, c, d = "żółty"] = colors;
console.log(d);  // "żółty"

// Zamiana wartości zmiennych
let x1 = 10, y1 = 20;
[x1, y1] = [y1, x1];
console.log(x1, y1);  // 20, 10
Destructuring tablic przypisuje elementy po pozycji. Przydatne przy odbieraniu wyników z funkcji zwracających tablice. Zamiana wartości to elegancki trick.

Destructuring obiektów


let user = { name: "Anna", age: 30, email: "anna@test.pl", role: "tester" };

// Bez destructuringu
let name1 = user.name;
let age1 = user.age;

// Z destructuringiem
let { name, age, email } = user;
console.log(name);   // "Anna"
console.log(email);  // "anna@test.pl"

// Zmiana nazwy zmiennej
let { name: userName, role: userRole } = user;
console.log(userName);  // "Anna"
console.log(userRole);  // "tester"

// Wartości domyślne
let { name: n, phone = "brak" } = user;
console.log(phone);  // "brak"

// Zagnieżdżony destructuring
let order = { customer: { name: "Anna", city: "Warszawa" } };
let { customer: { city } } = order;
console.log(city);  // "Warszawa"
Destructuring obiektów przypisuje po nazwie klucza (nie po pozycji jak w tablicach). Bardzo często używane w parametrach funkcji i przy pracy z API response.

Destructuring w funkcjach


// Bez destructuringu
function displayUser(user) {
  console.log(`${user.name}, wiek: ${user.age}`);
}

// Z destructuringiem w parametrach
function displayUser({ name, age, role = "user" }) {
  console.log(`${name}, wiek: ${age}, rola: ${role}`);
}

let user = { name: "Anna", age: 30, email: "anna@test.pl" };
displayUser(user);  // "Anna, wiek: 30, rola: user"

// Destructuring w callback z tablicą obiektów
let users = [
  { name: "Anna", score: 85 },
  { name: "Bartek", score: 92 }
];

users.forEach(({ name, score }) => {
  console.log(`${name}: ${score} pkt`);
});
Destructuring w parametrach funkcji to bardzo częsty pattern. Od razu widać, jakie właściwości obiektu są używane w funkcji. Pokaż też użycie w forEach callback.

Spread operator ...


// Łączenie tablic
let fruits = ["jabłko", "banan"];
let veggies = ["marchew", "brokuł"];
let all = [...fruits, ...veggies];
console.log(all);  // ["jabłko", "banan", "marchew", "brokuł"]

// Kopiowanie tablicy (płytka kopia)
let original = [1, 2, 3];
let copy = [...original];

// Łączenie obiektów
let defaults = { theme: "dark", lang: "pl", fontSize: 14 };
let userPrefs = { lang: "en", fontSize: 16 };
let settings = { ...defaults, ...userPrefs };
console.log(settings);
// { theme: "dark", lang: "en", fontSize: 16 }
// userPrefs nadpisuje defaults!

// Dodawanie właściwości
let user = { name: "Anna", age: 30 };
let extendedUser = { ...user, role: "tester", active: true };
Spread to trzy kropki. W tablicach "rozpakowuje" elementy, w obiektach "rozpakowuje" pary klucz-wartość. Przy łączeniu obiektów, późniejsze wartości nadpisują wcześniejsze (jak w settings).

Rest parameters ...args


// Rest w parametrach funkcji - zbiera pozostałe argumenty
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
console.log(sum(1, 2, 3));       // 6
console.log(sum(10, 20, 30, 40)); // 100

// Rest z innymi parametrami
function logFirst(first, ...rest) {
  console.log("Pierwszy:", first);
  console.log("Pozostałe:", rest);
}
logFirst("A", "B", "C", "D");
// Pierwszy: A
// Pozostałe: ["B", "C", "D"]

// Rest w destructuringu
let [head, ...tail] = [1, 2, 3, 4, 5];
console.log(head);  // 1
console.log(tail);  // [2, 3, 4, 5]

Spread vs Rest

Spread - rozpakowuje elementy: [...arr]
Rest - zbiera elementy: function(...args)

Rest i spread używają tej samej składni (trzy kropki), ale w odwrotnych kierunkach. Rest zbiera wiele wartości do jednej tablicy, spread rozkłada tablicę na wiele wartości.

JSON

JavaScript Object Notation

Ostatni blok na dziś - JSON. To format danych, który będzie kluczowy przy pracy z API.

Czym jest JSON?

JSON (JavaScript Object Notation) - tekstowy format wymiany danych

Obiekt JavaScript


let product = {
  name: "Szampon",
  price: 15.99,
  inStock: true,
  tags: ["higiena", "włosy"]
};

JSON (tekst)


{
  "name": "Szampon",
  "price": 15.99,
  "inStock": true,
  "tags": ["higiena", "włosy"]
}

Różnice

W JSON klucze muszą być w cudzysłowach. Brak komentarzy, brak undefined, brak funkcji.

JSON wygląda prawie jak obiekt JS, ale to STRING - tekst. Kluczowe różnice: podwójne cudzysłowy wokół kluczy, brak trailing comma, brak komentarzy.

JSON.stringify() - obiekt na tekst


let user = {
  name: "Anna",
  age: 30,
  skills: ["JS", "HTML", "CSS"]
};

// Konwersja obiektu na string JSON
let jsonString = JSON.stringify(user);
console.log(jsonString);
// '{"name":"Anna","age":30,"skills":["JS","HTML","CSS"]}'

console.log(typeof jsonString);  // "string"

// Ładne formatowanie (pretty print)
let pretty = JSON.stringify(user, null, 2);
console.log(pretty);
// {
//   "name": "Anna",
//   "age": 30,
//   "skills": [
//     "JS",
//     "HTML",
//     "CSS"
//   ]
// }
stringify zamienia obiekt JS na string JSON. Drugi argument to replacer (null = bez filtra), trzeci to liczba spacji dla wcięć. Pretty print jest przydatny do debugowania.

JSON.parse() - tekst na obiekt


let jsonString = '{"name":"Anna","age":30,"isActive":true}';

// Konwersja stringa JSON na obiekt
let user = JSON.parse(jsonString);
console.log(user.name);     // "Anna"
console.log(user.age);      // 30
console.log(typeof user);   // "object"

// Niepoprawny JSON - błąd!
try {
  let broken = JSON.parse("{ name: Anna }");  // brak cudzysłowów!
} catch (error) {
  console.log("Błąd parsowania:", error.message);
}

// Częsty pattern: parse + praca z danymi
let responseBody = '{"products":[{"name":"Szampon","price":15.99}]}';
let data = JSON.parse(responseBody);
console.log(data.products[0].name);  // "Szampon"
parse zamienia string JSON na obiekt JS. Niepoprawny JSON rzuca wyjątek - dlatego warto opakowywać w try/catch. To kluczowe przy pracy z odpowiedziami API.

JSON w kontekście API


// Typowa odpowiedź API (jako string JSON)
let apiResponse = `{
  "status": 200,
  "data": {
    "users": [
      { "id": 1, "name": "Anna", "role": "tester" },
      { "id": 2, "name": "Bartek", "role": "developer" }
    ],
    "total": 2
  }
}`;

// Parsowanie odpowiedzi
let response = JSON.parse(apiResponse);

// Praca z danymi
let users = response.data.users;
let testers = users.filter(u => u.role === "tester");
console.log("Testerzy:", testers.map(u => u.name));  // ["Anna"]

// Tworzenie body requestu
let newUser = { name: "Celina", role: "tester" };
let requestBody = JSON.stringify(newUser);
console.log(requestBody);
// '{"name":"Celina","role":"tester"}'
To jest kwintesencja tego dnia - połączenie JSON z tablicami i obiektami. W pracy testera API: odbieramy JSON (parse), przetwarzamy dane (filter, map), wysyłamy JSON (stringify). To fundament następnych dni szkolenia.
